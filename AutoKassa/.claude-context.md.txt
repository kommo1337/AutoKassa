# AutoKassa - Личная Касса

## Техническое Задание

### Описание проекта
Desktop приложение для учета личных финансов с возможностью:
- Записи доходов и расходов
- Категоризации операций
- Формирования отчетов
- Анализа финансов
- Планирования бюджета
- Защиты данных паролем

### Целевая аудитория
Физические лица, желающие вести учет личных финансов на локальном компьютере без облачных сервисов.

### Основные требования
- Локальное хранение данных (SQLite)
- Защита паролем
- Простой и понятный интерфейс
- Быстрая работа
- Наглядная визуализация (графики, диаграммы)
- Экспорт отчетов

## Архитектурные решения

### MVVM Pattern
- **Model** - сущности БД (Transaction, Category)
- **ViewModel** - бизнес-логика, команды (TransactionsViewModel)
- **View** - XAML разметка (TransactionsView.xaml)
- Связь через DataBinding и ICommand

### Dependency Injection
Все сервисы регистрируются в `App.xaml.cs`:
```csharp
services.AddScoped<ITransactionService, TransactionService>();
services.AddScoped<ICategoryService, CategoryService>();
services.AddScoped<IReportService, ReportService>();
// ...
```

### Навигация
Через `INavigationService` в `MainWindowViewModel`:
```csharp
_navigationService.NavigateTo<TransactionsViewModel>();
```

### Команды
Через `RelayCommand`:
```csharp
AddCommand = new RelayCommand(_ => AddTransaction());
```

## Стандарты кодирования

### Именование
- **PascalCase** - классы, методы, свойства, константы
- **camelCase** - параметры методов, локальные переменные
- **_camelCase** - private поля класса
- **Async** суффикс - для асинхронных методов

### Комментарии
- XML комментарии для public API
- `/// <summary>` для описания
- Inline комментарии для сложной логики

### Async/Await
Все операции с БД - асинхронные:
```csharp
public async Task<List<Transaction>> GetAllAsync()
{
    return await _context.Transactions.ToListAsync();
}
```
