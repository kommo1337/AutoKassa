# TASK: Переработка дизайна — Касса автосервиса

## Контекст проекта
Приложение-касса для автосервиса. Позволяет вести операции, категории, просматривать дашборд и управлять настройками.

## Стек
- **UI:** WPF (.NET), паттерн MVVM
- **ORM:** Entity Framework Core
- **БД:** SQLite (изучи схему самостоятельно из файла описания проекта и `.db` файла)
- **Биндинги:** INotifyPropertyChanged, ICommand

---

## Структура экранов
| Экран | Описание |
|---|---|
| Dashboard | Сводка по операциям, статистика |
| Категории | Управление категориями доходов/расходов |
| Операции | Список и добавление финансовых операций |
| Настройки | Параметры приложения |

---

## Новая фича: Разделение счетов (Наличный / Безналичный)

### Суть
Каждая операция теперь должна относиться к одному из двух счетов:
- 💵 **Наличный** (cash)
- 💳 **Безналичный** (non-cash / card)

### Изменения в БД
- [ ] Добавить поле `PaymentType` (enum или string: `Cash` / `NonCash`) в таблицу операций
- [ ] Создать EF миграцию
- [ ] Дефолтное значение для существующих записей — `Cash`

### Изменения в Model
- [ ] Добавить свойство `PaymentType` в модель операции
- [ ] Создать enum `PaymentType { Cash, NonCash }` (если не существует)

### Изменения в ViewModel
- [ ] Dashboard VM: считать итоги отдельно по наличному и безналичному счёту
- [ ] Операции VM: добавить выбор типа счёта при создании/редактировании операции
- [ ] Фильтрация операций по типу счёта (если предусмотрена макетом)

### Изменения в View (XAML)
- [ ] Отображение баланса по каждому счёту на Dashboard
- [ ] Переключатель Cash / NonCash при добавлении операции
- [ ] Визуальное разделение или иконка типа счёта в списке операций

---

## Правила работы

1. **Соблюдай MVVM строго** — никакой бизнес-логики в code-behind, только биндинги
2. **После каждого изменённого экрана — останавливайся и жди подтверждения**
3. **Сначала БД и модели, потом VM, потом View** — не наоборот
4. **Не трогай файлы, не относящиеся к текущей задаче**
5. **Перед началом работы** — изучи структуру проекта, схему БД, все файлы из `/ClaudeDesign`
